# 計画ルール

- プランは目的・スコープ・非ゴール・受け入れ基準・ベリファイプラン・テストプラン・リスク・エビデンスを定義すること。
- **ベリファイプラン**：静的解析チェック・スペック適合基準・ドキュメントドリフトチェック・取得するエビデンスを網羅する。
- **テストプラン**：ユニットテスト・統合テスト・回帰テスト・エッジケース・取得するエビデンスを網羅する。
- プランは低レベルの実装を早期に凍結せずに、成果物と契約を制約すること。
- タスクが進行中は進捗チェックリストを更新し続けること。
- 元のプランからの意味のある逸脱は、サイレントにドリフトするのではなく記録すること。
