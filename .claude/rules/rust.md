# Rust ルール

このルールは Rust プロジェクトでのみアクティブになる。

## エラーハンドリング

- `unwrap()` と `expect()` は本番コードで使わない（テストは例外）
- `?` 演算子でエラーを伝播させる
- `anyhow` または `thiserror` でエラー型を定義する
- `panic!` は真に回復不可能な状態のみ

## 所有権とライフタイム

- 所有権を移動させるより借用（`&`）を優先する
- ライフタイム注釈はコンパイラが推論できない場合のみ明示する
- `Rc<RefCell<T>>` の多用はアーキテクチャ設計を見直すシグナル

## 非同期

- `async/await` を使う場合は `tokio` または `async-std` を一貫して使う
- `block_on` は非同期コンテキストの最外層のみ
- `Send + Sync` トレイト境界を意識する

## コード品質

- `clippy` の警告をゼロにする（`#[allow(...)]` は理由コメント付きで）
- `rustfmt` でフォーマットする（CI で強制）
- `unsafe` ブロックには安全性の不変条件をコメントで記述する
- マジックナンバーには `const` を使う

## テスト

- `#[cfg(test)]` モジュールでユニットテストを書く
- `tests/` ディレクトリに統合テストを置く
- `cargo test` がすべてパスすること

## 検証

```sh
./scripts/verify-rust.sh
```
