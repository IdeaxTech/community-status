## 目的

月単位のカレンダービューを追加し、毎週木曜日のもくもく会と追加イベントを一覧できるようにする。誰でも特定日付にイベントを追加でき、その月に何があるか一目でわかるようにする。

## スコープと非ゴール

**スコープ**:
- 月次カレンダー表示（前月・次月ナビゲーション付き）
- 毎週木曜日のもくもく会を自動表示（静的ロジックで生成）
- 任意のイベントを特定日付に追加できる（認証不要）
- イベント一覧を日付セルに表示
- SQLite の `calendar_events` テーブルで永続化
- API: `GET /api/calendar?year=&month=`、`POST /api/calendar`

**非ゴール**:
- イベントの編集・削除
- 繰り返しイベントのカスタム設定
- 外部カレンダー（Google Calendar 等）との同期
- 通知・リマインダー

## 前提

- 既存の SQLite DB（`src/lib/db.ts`）に `calendar_events` テーブルを追加する
- もくもく会の定期開催情報（木曜 13:00〜20:00）はコード上の定数として管理
- 外部カレンダーライブラリは使わず、Tailwind CSS のグリッドで実装する

## 受け入れ基準

- [ ] 月次カレンダーが表示される（7列グリッド）
- [ ] 毎週木曜日のセルに「もくもく会 13:00〜20:00」が自動表示される
- [ ] 前月・次月へのナビゲーションができる
- [ ] 日付セルをクリックしてイベントタイトルを入力・追加できる
- [ ] 追加したイベントが該当日付のセルに表示される
- [ ] `GET /api/calendar?year=2026&month=6` でその月のイベント一覧が返る
- [ ] `POST /api/calendar` でイベントを追加できる（`date`, `title` 必須）
- [ ] title が空・date が不正の場合は 400 を返す

## 設計決定

**もくもく会の自動表示**: DB には保存せず、クライアント側でその月の木曜日を計算して表示する。DB をシンプルに保ち、開催曜日・時間変更時はコード修正で対応。

**カレンダーライブラリなし**: `date-fns` や `react-calendar` を使わず、日付計算を素の `Date` API で実装。依存最小化のため。

## 実装概要

1. **DB 層拡張** (`src/lib/db.ts`) — `calendar_events(id, date TEXT, title TEXT, created_at TEXT)` テーブルを `initSchema` に追加・`getCalendarEvents(year, month)` / `addCalendarEvent(date, title)` を追加
2. **API ルート** (`src/app/api/calendar/route.ts`) — `GET`: year/month クエリパラメータでイベント取得 / `POST`: date + title でイベント追加
3. **CalendarView コンポーネント** — 月グリッド・木曜自動表示・日付クリックで追加フォーム
4. **MainPage 統合** — 既存コンポーネントの下に `<CalendarView />` を追加
