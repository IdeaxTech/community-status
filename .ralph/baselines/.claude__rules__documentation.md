# ドキュメントルール

## 原則

- ドキュメントはコードと同期して更新する
- WHY（なぜ）を文書化する — WHAT（何を）はコードが語る
- コメントは非自明な制約・微妙な不変条件・特定のバグへの回避策のみ
- CLAUDE.md を短く安定した状態に保つ

## 構造

- 常時参照ガイダンス → `CLAUDE.md`
- トピック別詳細 → `.claude/rules/<topic>.md`
- ワークフロー → `.claude/skills/<skill>/SKILL.md`
- 進行中の作業 → `docs/plans/active/`
- 完了した作業 → `docs/plans/archive/`
- レビュー・検証・テスト成果物 → `docs/reports/`

## 更新が必要な場合

- API またはインターフェースが変わった
- ワークフローまたはコマンドが変わった
- 重要なアーキテクチャ決定が行われた
- 繰り返す間違いが発見された（フック・ルール・CI に昇格させる）
